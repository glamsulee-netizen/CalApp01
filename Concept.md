# PWA Веб-приложение для бронирования записей к специалисту

## Общая задача

Создать многопользовательскую платформу для мастеров, оказывающих услуги, и их клиентов. Платформа позволяет мастерам создавать календари с доступными слотами для записи, а клиентам - легко находить и бронировать удобное время.

**Цель**: Максимально упростить процесс проверки и бронирования свободных временных слотов для записи к мастеру с минимальным количеством действий.

## Ключевые особенности

### Безопасность и масштабируемость
- Безопасная архитектура для защиты данных пользователей
- Код и архитектура, легко дорабатываемые с помощью ИИ
- Возможность добавления новых функций после запуска

### Мультиязычность
- Заложена возможность реализации многоязычного интерфейса

### Уведомления
- Push-уведомления для всех ролей:
  - Мастерам: когда клиент записывается
  - Клиентам: если мастер отменяет/переносит бронь
  - Админам: при запросе на включение функции календаря или новой оплате
- Автонапоминания о предстоящих событиях (за 24 часа и 2 часа, настраиваемые)

## Технологический стек

### Вариант 1: Классический подход
- **Frontend**: React 18 + Vite + TypeScript (PWA)
- **Backend**: Node.js + Express + TypeScript
- **База данных**: PostgreSQL 15 + Prisma ORM
- **Кеш**: Redis 7
- **Real-time**: Socket.IO
- **Push-уведомления**: Web Push API (VAPID)
- **Деплой**: Docker + Nginx + Let's Encrypt

### Вариант 2: Современный подход (рекомендуется)
- **Frontend**: Next.js 14 + TypeScript + Tailwind CSS + PWA
- **Backend**: Node.js + Fastify (быстрее Express, отличная TypeScript-поддержка)
- **База данных**: PostgreSQL + Prisma ORM
- **Аутентификация**: NextAuth.js v5 (magic link + credentials)
- **Real-time**: Socket.io (обновления календаря в реальном времени)
- **Email**: Nodemailer + SMTP или Resend API
- **Push-уведомления**: web-push (VAPID)
- **Деплой**: Docker Compose на VPS + Nginx reverse proxy + Certbot SSL
- **Процессы**: PM2 или внутри Docker
- **Cron**: node-cron (напоминания)

## Основные процессы и логика

### Роли пользователей
1. **Клиент** - пользователь, который записывается к мастеру
2. **Мастер** - пользователь, оказывающий услуги и управляющий своим календарем
3. **Админ** - администратор платформы

### Путь клиента
1. Клиент получает ссылку на календарь мастера
2. Переходит по ссылке → открывается страница с календарем и свободными слотами
3. Может сохранить страницу на домашний экран телефона (PWA)
4. **Без авторизации**: видит свободные слоты, но не может записаться
5. **С авторизацией**: может бронировать слоты

### Авторизация
- Неавторизованный пользователь: видит плашку "Авторизоваться" при нажатии на слот
- Авторизованный пользователь: видит плашку "Записаться"
- Внизу интерфейса: кнопка входа в личный кабинет / авторизации

### Личный кабинет пользователя
- **Для клиентов**:
  - Список забронированных слотов в календарях мастеров
  - Избранные календари для быстрого доступа
  - Кнопка "создать календарь" (запрос на включение интерфейса мастера)

- **Для мастеров** (после одобрения админом):
  - Управление своим календарем
  - Копирование ссылки на календарь
  - Просмотр и управление бронями от клиентов

### Процесс становления мастером
1. Пользователь нажимает "создать календарь" в ЛК
2. Админу отправляется запрос на включение интерфейса мастера
3. Админ рассматривает запрос и включает доступ на определенное время
4. В будущем: автоматизация через оплату (пользователь сам выбирает срок доступа)

### Интерфейс календаря
- **Колонки**: дни недели
- **Шапка колонки**: буква дня недели + число дня месяца
- **Текущий день**: выделен фоном в шапке
- **Строки**: часы (настраиваемый интервал, по умолчанию 8:00-21:00)
- **Навигация**: свайп влево/вправо для листания недель

### Управление слотами (для мастеров)
- Создание слота: одним нажатием (1 час по умолчанию)
- Изменение длительности: перетаскивание нижней границы с шагом 15 минут (как в Google Calendar)
- Перемещение слота: жестами на другое время/день

### Начальный экран (лендинг)
- Для неавторизованных пользователей, зашедших без указания календаря
- Простой лаконичный лендинг с описанием проекта
- Пример заголовка: "Здесь можно записаться к любимому специалисту на услугу. Просто попросите у него ссылку"
- Кнопки "Регистрация/Вход"

## Дизайн и UX требования
- Минималистичный интерфейс с продуманным дизайном
- Приоритет по дизайну и эргономике для пользователей iPhone
- Совместимость с Android
- Разные интерфейсы для разных ролей:
  - Клиентский интерфейс: максимально улучшен для просмотра календаря и бронирования
  - Интерфейс мастера: оптимизирован для управления календарем и бронями
  - Интерфейс админа: управление доступом к функциям

## Архитектурные принципы
1. **Модульность**: компоненты и сервисы легко заменяемы
2. **Масштабируемость**: поддержка роста числа пользователей и мастеров
3. **Безопасность**: защита данных, валидация входных данных, безопасные API
4. **Производительность**: оптимизация загрузки, кеширование, эффективные запросы
5. **Поддержка ИИ**: кодовая база, удобная для доработки с помощью ИИ-инструментов
