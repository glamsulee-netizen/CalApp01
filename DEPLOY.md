# CalApp01 — Инструкция по деплою (Автоматизированная)

Теперь деплой платформы сведен к минимуму благодаря встроенной автоматизации (автоматическое получение SSL через Caddy и скрипт первоначальной настройки `setup.sh`).

## Требования к VPS

- **ОС**: Ubuntu 22.04+ или Debian 11+
- **RAM**: минимум 1 GB (рекомендовано 2 GB)
- **Диск**: 10 GB+
- Доменное имя (A-запись в DNS должна указывать на IP вашего сервера)

---

## Часть 1. Первоначальный деплой на чистый сервер

Вам нужно зайти на ваш сервер по SSH всего один раз!

### 1. Подключение к серверу
Откройте терминал на вашем компьютере и подключитесь к серверу:
```bash
ssh root@ВАШ_IP_АДРЕС
```

### 2. Клонирование репозитория
```bash
git clone <URL-вашего-репозитория> ~/calapp
cd ~/calapp
```

### 3. Запуск автоматической установки
Сделайте скрипт исполняемым и запустите его:
```bash
chmod +x setup.sh
./setup.sh
```

**Скрипт сделает всё за вас:**
1. Установит Docker.
2. Спросит ваш домен и email.
3. Сгенерирует надежные пароли для базы данных, JWT и ключи VAPID (для Push-уведомлений).
4. Настроит `.env` файл.
5. Запустит проект с автоматическим HTTPS сертификатом.

По завершении скрипт выдаст вам реквизиты администратора (Email и сгенерированный пароль) для первого входа.

## Часть 2. Автоматическое обновление (CI/CD)

Чтобы проект обновлялся сам при каждом `git push` в ветку `main`, мы добавили поддержку GitHub Actions.
Вам не нужно больше заходить на сервер руками для применения обновлений кода!

### Настройка GitHub Actions
Перейдите в репозиторий вашего проекта на GitHub → **Settings** → **Secrets and variables** → **Actions**, и добавьте 3 новых **Repository secret**:

1. `SERVER_HOST` — IP адрес вашего сервера.
2. `SERVER_USER` — Имя пользователя сервера (например, `root` или `ubuntu`).
3. `SERVER_SSH_KEY` — Ваш приватный SSH ключ для доступа к серверу.
   *(Если вы подключаетесь по паролю, вам нужно будет сначала сгенерировать SSH ключ на вашем компьютере, добавить публичный ключ на сервер в `~/.ssh/authorized_keys`, а приватный ключ скопировать сюда).*

Как только эти секреты будут добавлены, любое изменение в ветке `main` будет автоматически загружаться на сервер и перезапускать контейнеры.

## Часть 3. Ручное управление и Бэкапы

Все базовые операции теперь выполняются через Docker Compose из папки `~/calapp`.

**Просмотр логов:**
```bash
cd ~/calapp
docker compose logs -f server
```

**Принудительный перезапуск:**
```bash
docker compose restart
```

**Бэкап базы данных:**
```bash
docker compose exec postgres pg_dump -U calapp calapp_db > backup_$(date +%Y%m%d).sql
```
